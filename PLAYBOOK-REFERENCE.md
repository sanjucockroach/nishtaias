# Immersive Website Playbook — Local Adaptation

Source reviewed in full on 2026-07-17. This project applies the playbook with the client-specific instruction that the hero must remain professional and free of decorative 3D.

## Working rules

1. Build mobile-first and verify at 390px and desktop widths.
2. Commit to one accent, one radius grammar, one depth philosophy, and one motion timing.
3. Use negative display tracking and semantic, accessible HTML.
4. Keep touch targets at least 44px and respect `prefers-reduced-motion`.
5. Use 3D as subtle UI depth—perspective panels, layered surfaces, dashboard objects—not as a hero spectacle.
6. Complete SEO metadata, structured data, sitemap, robots, manifest, and favicon.
7. Run lint, type checking, build, and browser golden-path checks before publishing.
8. Iterate plan → act → observe → critique → refine until the quality gate is met.
